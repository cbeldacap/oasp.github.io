Open Application Standard Platform

  • originates from the APPS Evolve!
  • meets Industrialization's expectations
  • is hosted at GitHub
  • has three variants:
    • Java (OASP4J)
    • .Net (OASP4NET)
    • JavaScript (OASP4JS)

JavaScript Platform

  • has its own repositories and pages at GitHub
  • focuses only on the client at the moment
  • is going to provide:
    • an application template (aka project seed)
    • AngularJS extensions which address typical problems
    • a sample application built on top of the template showing the extensions working

How is the work organized?

  • contributors needs to have a GitHub account -> a photo and a real name is required
  • tasks and discussions around them are organized in the form of GitHub's issues
  • at the moment we have two branches: main and development
  • we are going to apply the social coding approach:
    • contributors fork the oasp4js repository and clone it
    • once their changes are ready (committed and pushed), they issue a pull request
    • if the pull request is accepted, the changes are merged into the development branch
    • stable releases are on the main branch
  • The Continues Integration environment is set up in the cloud (Microsoft Azure / SkySight) at http://oasp-ci.cloudapp.net/

How do I set up my development environment?

  1. install Node.js, Grunt, and Bower; for details refer to the GitHub's Wiki
  2. download or fork the oasp4js-sample repository
  3. open a command prompt and go to the application’s root directory
  4. type npm install; it installs modules needed by Grunt and (as a post install task) calls bower install which, in turn, installs application’s dependencies
  5. open the project under your IDE - WebStorm or IntelliJ preferred :)

How is the JavaScript client integrated with the Java backend?

  • JavaScript client
    • under /java there is pom.xml which runs the build using Grunt and creates a Maven module (packaging ZIP)
    • mvn clean install installs the ZIP module containing the JavaScript client in the local repository
  • Java backend
    • the oasp4j-example-application has a dependency to the ZIP module containing the JavaScript client
    • the dependency is active only when built with jsclient profile
    • the maven-war-plugin is configured so that the JS client module overlays the main WAR dependency

How does the current application skeleton work?

  • the JS client requires providing REST services by the backend
  • the JS client itself may be served by the backend
  • another scenario would be to deploy the JS client e.g. on the Apache httpd which redirects REST services' calls to the backend
